# 运行测试

##  配置检查

当您完成配置文件后，可以在**项目的根目录**运行以下命令以检查配置文件的正确性：

```shell
fuzzx validate
```

这条命令会检查您的`fuzzx.yaml`文件是否缺少了关键字段，并给您相应的提示。


如果您觉得一切已经就绪了，可以运行以下命令进行构建(在项目的根目录下执行)：

```shell
fuzzx build
```


##  Fuzz ！

当您把一切都准备好了，下面就可以把代码推送到`FuzzX`了！
我们推荐您将代码提交到您的`GitHub`仓库。


提交完代码，请移步[FuzzX主页](http://fuzzx.guardstrike.com)。
